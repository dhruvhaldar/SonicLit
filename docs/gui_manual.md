# GUI User Manual

SonicLit provides two graphical user interfaces (GUIs) to interact with the library's features without writing code: a **Desktop App** (Tkinter) and a **Web App** (Streamlit).

## 1. Desktop Application

The desktop application is built using Tkinter and provides a native window experience.

### Launching the App
To launch the desktop GUI, run the following command from the root of the repository:

```bash
python scripts/run_desktop.py
```

### FWH Solver Tab
This tab allows you to run the Ffowcs-Williams Hawkings (FWH) solver for aeroacoustics.

*   **Surface File Prefix**: The path to your surface data files. The solver expects files named in the format `<prefix>0.csv`, `<prefix>1.csv`, etc., and `<prefix>Avg.csv`.
    *   *Example*: If your files are `tests/data/surf_0.csv`, `tests/data/surf_1.csv`, enter `tests/data/surf_`.
    *   *Tip*: Use the "Browse" button to select the `0.csv` file, and the prefix will be automatically extracted.
*   **Output File Prefix**: The path where output files will be saved.
    *   *Example*: `results/fwh_out` will generate `results/fwh_out0.csv` (for the first observer), `results/fwh_out1.csv`, etc.
*   **Observer Locations**: A list of lists representing Cartesian coordinates `[x, y, z]` for each observer.
    *   *Format*: `[[x1, y1, z1], [x2, y2, z2], ...]`
    *   *Example*: `[[0.0, 0.0, 1.0]]`
*   **Time Step (dt)**: The time interval between snapshots in your surface data (seconds).
*   **Number of Steps**: The number of time steps (snapshots) to process.
*   **Mach Number**: The freestream Mach number vector `[Mx, My, Mz]`.
    *   *Example*: `[0.1, 0.0, 0.0]` for flow in x-direction.
*   **Temperature**: Freestream temperature in Kelvin.
*   **Permeable Surface**: Check this box if the integration surface is permeable (e.g., for off-body porous FWH).

### Spectral Analysis Tab
This tab allows you to perform signal processing on time-series data.

*   **Signal CSV File**: Select a CSV file containing your signal data.
*   **Time Column Name**: The header name of the column containing time values.
*   **Signal Column Name**: The header name of the column containing the signal values (e.g., pressure).
*   **Method**:
    *   **FFT**: Performs a simple Fast Fourier Transform.
    *   **Welch**: Estimates Power Spectral Density (PSD) using Welch's method (averaging periodograms).
*   **Plot Spectrum**: Click to generate and display the spectrum plot.

## 2. Web Application

The web application is built using Streamlit and runs in your browser. It is suitable for both local use and deployment on a server.

### Launching the App
To launch the web GUI, run:

```bash
./scripts/run_web.sh
```
Or directly via Streamlit:
```bash
streamlit run src/soniclit/gui/web/app.py
```

### FWH Solver Tab
*   **Upload Surface Data**: Upload a **ZIP file** containing your surface CSV files (`Avg.csv`, `0.csv`, `1.csv`...). The app will extract them to a temporary location for processing.
*   **Configuration**: Similar inputs to the Desktop app (Observer Locations, Mach Number, etc.).
*   **Results**:
    *   After running, you can download all result CSVs as a single ZIP file.
    *   Preview plots (PNGs) generated by the solver are displayed directly in the browser.

### Spectral Analysis Tab
*   **Upload Signal CSV**: Upload a single CSV file.
*   **Interactive Configuration**: Select columns from dropdown menus (automatically populated from the CSV headers).
*   **Interactive Plot**: The generated plot is displayed using Matplotlib and updates automatically when parameters change.
